# 🚀 Client Onboarding Pipeline

> Contract signed → folder created → welcome email sent → CRM updated → invoice generated → team notified → tasks assigned → credentials delivered. Fully automatic. Under 60 seconds.

## The problem

Every time a new client signs on, someone manually:

- Creates a folder structure in Drive
- Sends a welcome email
- Adds the client to the CRM
- Generates a proforma invoice
- Pings the team on Slack
- Creates onboarding tasks in the project tool
- Sends access credentials

That's 30–60 minutes of repetitive admin work per client. This pipeline does all of it in under 60 seconds — triggered by a single webhook when the contract is signed.

---

## Pipeline steps

| Step | What happens | Tools |
|---|---|---|
| 1 | Contract parsed by AI — extracts client name, service, value, start date | AI / LLM, PDF parser |
| 2 | Google Drive folder created with full subfolder structure | Drive API |
| 3 | Personalised welcome email sent with project brief + scheduling link | Gmail API + AI |
| 4 | CRM record created — contact, deal, stage: Onboarding | HubSpot / Pipedrive API |
| 5 | Proforma invoice generated as PDF and sent to client | ReportLab, Gmail API |
| 6 | Team notified in Slack #new-clients with client details | Slack Webhooks |
| 7 | Onboarding tasks created in Asana and assigned to team | Asana API |
| 8 | Access credentials sent securely to client email | Gmail API |

Total time: **~45 seconds**. Manual time saved: **~45 minutes per client**.

## Production Setup

```bash
git clone https://github.com/yourusername/onboarding-demo
cd onboarding-demo
pip install -r requirements.txt
cp .env.example .env
# Fill in all required API keys (see .env.example)
python onboarding.py --trigger webhook  # listen for contract webhooks
# or
python onboarding.py --client "Acme Corp" --email ceo@acme.com --service "AI Automation"
```

### Trigger methods

**Webhook** (recommended) — pipe from your contract signing tool (DocuSign, HelloSign, PandaDoc):
```bash
python server.py --port 8000
# POST /webhook/contract-signed  {"client_name": "...", "email": "...", "service": "..."}
```

**Manual trigger** — run for a specific client:
```bash
python onboarding.py --client "Nova Digital" --email hello@nova.pl --service "Chatbot setup" --value 2400
```

**CSV batch** — onboard multiple clients at once:
```bash
python onboarding.py --batch clients.csv
```

## Configuration

Edit `config.py` to match your business:

```python
# Folder structure created in Drive
DRIVE_FOLDERS = ["Contracts", "Invoices", "Deliverables", "Communications", "Assets"]

# Asana tasks created for every new client
ONBOARDING_TASKS = [
    {"name": "Kickoff call", "due_days": 2, "assignee": "ops@yourcompany.com"},
    {"name": "Requirements document", "due_days": 5},
    {"name": "Environment setup", "due_days": 7},
    {"name": "First delivery", "due_days": 14},
    {"name": "Client review", "due_days": 16},
    {"name": "Sign-off", "due_days": 18},
]

# Slack channel for notifications
SLACK_CHANNEL = "#new-clients"

# CRM to use
CRM = "hubspot"  # or "pipedrive"

# Invoice settings
INVOICE_PAYMENT_TERMS = 14  # days
INVOICE_CURRENCY = "EUR"
```

## Customisation examples

**Different folder structures per service type:**
```python
FOLDER_TEMPLATES = {
    "chatbot": ["Docs", "Training Data", "Deployments", "Reports"],
    "automation": ["Scripts", "Configs", "Logs", "Docs"],
    "reporting": ["Data", "Reports", "Dashboards", "Archive"],
}
```

**AI-personalised welcome emails:**
The welcome email is generated by AI based on the client's industry, service type, and any notes from the contract — not a generic template.

**Multi-language support:**
Pipeline detects client language from contract and sends all communications in the appropriate language (PL/EN/DE supported out of the box).
